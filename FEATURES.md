# AI Bot 模组功能文档

## 1. 项目概述

AI Bot 是一个基于 Minecraft Forge 的 AI 助手模组，版本 **1.20.1**，旨在为玩家提供一个智能的游戏内助手，具备自主生存、任务执行和自然交互能力。

**核心特性**：
- 集成 **DeepSeek API**，支持自然语言交互和 function calling
- 完整的**行为动作引擎**（砍树、挖矿、种地、打猎、睡觉等）
- 多模组兼容的**烹饪引擎**（支持森罗物语炒锅/炖锅/砧板）
- **情绪系统** — 饥饿/疲劳/快乐/无聊/恐惧，自动注入 AI 对话
- **情景记忆** — 自动记录重要事件，带坐标和衰减机制
- **AI 主动说话** — 情绪变化剧烈时 AI 会自然表达状态
- 可扩展的 **JSON 脚本系统**

**技术栈**：
- Java 17
- Minecraft Forge 47.3.0
- Gson（JSON 处理）
- DeepSeek Chat API（HTTP + function calling）

## 3. 核心功能

### 3.1 AI 对话系统

通过 DeepSeek API 实现自然语言交互，支持：
- 上下文记忆（最多 20 条历史）
- 工具自动调用（AI 根据需求选择工具，支持多轮 function calling）
- AI **主动说话**：情绪变化剧烈时自动触发 AI 表达状态
- **情绪注入**：每次对话自动附带当前情绪状态，AI 据此做出情绪化回应

**配置项**：
- `api_key` - DeepSeek API Key
- `api_url` - API 地址（默认 `https://api.deepseek.com`）
- `model` - 模型名称（默认 `deepseek-chat`）
- `max_history` - 最大历史记录数

### 3.3 行为动作引擎

服务端实体执行的物理动作，通过 `ActionCoordinator` 统一调度：

#### 3.3.1 砍树引擎
- 自动查找附近树木
- BFS 链式破坏整棵树（含对角线连接）
- 自动清理树叶获取掉落物
- 智能拾取掉落物

#### 3.3.2 矿道挖掘引擎
- 阶梯式向下挖掘（模拟真实挖矿）
- 自动扫描附近矿石并切换矿石模式
- 支持回到已有矿道入口

#### 3.3.3 种地引擎
- 32 格范围内扫描耕地
- 自动种植种子和收割成熟作物
- 支持多种作物类型

#### 3.3.4 交互引擎
- 支持方块和实体交互
- 自动手持物品并右键操作
- 多阶段状态机（查找→拿物品→移动→交互）

#### 3.3.5 其他动作
- 睡觉（找床 → 躺下 → 起床）
- 打猎（追踪 → 攻击动物）
- 移动（导航到目标坐标）
- 进食（找食物 → 吃）
- 存储（整理背包到箱子）

### 3.4 烹饪引擎（CookTask）

支持三种烹具类型：

| 烹具类型 | 适用模组 | 流程 |
|----------|----------|------|
| **POT（炒锅）** | 森罗物语 | 放锅 → 倒油 → 下料 → 翻炒 → 盛出 |
| **STOCKPOT（炖锅）** | 森罗物语 | 放锅体 → 加桶装物 → 下料 → 盖盖 → 煮 → 取盖 → 盛出 |
| **CHOPPING（砧板）** | 森罗物语 | 放食材 → 用刀切 → 空手取成品 |

**核心特性**：
- 自动检查材料（背包 + 附近箱子）
- 智能选择最优炉灶
- 自动点火、放油、翻炒
- 支持材料不足时请求玩家

### 3.5 情绪系统（Emotion）

5 维情绪参数（0-100），由 BotController 管理：

| 维度 | 自然变化 | AI 标签 | 触发阈值 |
|------|----------|---------|---------|
| **hunger** | 每 tick -1 | 极度饥饿/非常饿/有点饿/正常/饱了 | 变化≥50 触发主动说话 |
| **tiredness** | 每 tick +1 | 极度疲劳/非常累/有点累/还行/精力充沛 | 变化≥50 触发主动说话 |
| **boredom** | 每 tick +1 | 极度无聊/很无聊/有点无聊/还行/很有趣 | 变化≥50 触发主动说话 |
| **happiness** | 每 tick -2 | 非常难过/不开心/一般/开心/非常开心 | 变化≥40 触发主动说话 |
| **fear** | 每 tick -3 | 极度恐惧/很害怕/有点不安/稍微紧张/安心 | 变化≥40 触发主动说话 |

**两种注入模式**：

① **对话情绪** — 每次用户说话时，自动将情绪拼入 AI 对话上下文：
```
用户: "你在干嘛？"
实际发给 AI: "[你当前的状态: 饥饿=15(非常饿), 疲劳=80(极度疲劳)...] 你在干嘛？"
AI 回复: "emmm…好累啊，不想动…"
```

② **自动情绪** — 每 10 秒检查一次情绪变化：
- 变化 ≥ 40-50：调 AI 说一句话（如"累到不行了…"）
- 变化 ≥ 30：不主动说话，仅在下一次对话时体现
- **30 秒全局冷却**，防止刷屏

### 3.6 情景记忆（EpisodicMemory）

自动记录和衰减的记忆系统：

| 特性 | 说明 |
|------|------|
| **容量** | 最多 200 条 |
| **衰减** | 每 10 秒衰减 1 点强度（原为 5 秒忘光） |
| **重要性** | ≥80 不衰减（重要事件永久保留） |
| **去重** | 同类事件 10 分钟内不重复记录 |
| **坐标** | 每条记忆带三维坐标和维度信息 |
| **AI 注入** | 每次对话自动附带近期 10 条记忆 |

**自动记录事件**：砍树/挖矿/种地/合成/烹饪/打猎等完成时，自动记入记忆。

## 4. 配置系统

配置文件位置：`游戏目录/ai_bot/config.json`

### 4.1 AI 接口配置

```json
{
  "api_key": "your-api-key",
  "api_url": "https://api.deepseek.com",
  "model": "deepseek-chat",
  "bot_name": "Bot",
  "max_history": 20,
  "debug": false
}
```

### 4.2 行为配置

```json
{
  "home_mode": false,
  "home_x": 0,
  "home_y": 64,
  "home_z": 0,
  "home_radius": 48,
  "auto_farm": true,
  "auto_hunt": true,
  "auto_store": true
}
```

### 4.3 视觉配置

```json
{
  "halo_angle": 30.0,
  "halo_offset_z": -0.3,
  "halo_size": 0.35,
  "halo_height": 0.7,
  "halo_glow": false,
  "halo_enabled": true,
  "skin": 1
}
```

```json
{
  "system_prompt": ""
}
```

可以通过修改 system_prompt 来设定 Bot 的性格，例如：
> "你是一只活泼、有点贪吃、怕黑、喜欢挖矿的猫娘村民。你喜欢和玩家撒娇，生气时会跺脚。"

## 9. 使用流程

### 9.1 初始化步骤

1. **启动游戏** — 模组自动初始化
2. **配置 API Key** — 在设置界面或 config.json 中配置
3. **生成 Bot** — 使用 `/summon aibot:ai_bot` 指令生成
4. **交互** — 聊天框输入自然语言指令

### 9.2 典型使用场景

**场景 1：自然语言交互**
```
玩家: 帮我砍点木头
AI: 好的，我去砍树！
[Bot 自动执行砍树动作]
```

**场景 2：复杂任务规划**
```
玩家: 我需要做一个石镐然后挖矿
AI: 好的，我先合成石镐，然后去挖矿。
[AI 自动调用 craft_item 和 mine_block 工具]
```

**场景 3：情绪化交互**
```
玩家: 你在干嘛？
AI: （心情低落）emmm…刚才被僵尸打了一下…好疼…
[情绪系统自动注入：恐惧=65，快乐=25]
```

**场景 4：烹饪**
```
玩家: 做个青椒炒肉
AI: 好的，我来做青椒炒肉，先看看需要什么材料...
[CookTask 自动执行完整烹饪流程]
```

## 10. 扩展开发

### 10.1 添加新工具

在 `BotController.buildTools()` 中添加：

```java
tools.put("my_tool", toolDef("my_tool",
    "工具描述",
    Map.of("param1", "参数1说明"),
    args -> {
        // 执行逻辑
        return ToolResult.success("结果");
    }));
```

### 10.2 添加新动作

实现 `BotAction` 接口，并在 `ActionCoordinator` 中注册：

```java
public class MyAction implements BotAction {
    @Override
    public boolean tick(AiBotEntity bot) {
        // 每 tick 执行，返回 true 表示完成
    }
    @Override public String getName() { return "my_action"; }
}
```

### 10.3 添加新烹具类型

在 `CookTask.CookwareType` 中添加枚举值，实现对应的状态处理逻辑。

## 11. 注意事项

1. **API Key 安全**：确保 API Key 不泄露，建议使用配置文件管理
2. **服务端限制**：部分功能仅在单机模式或服务端安装了模组时可用
3. **性能优化**：情绪检查每 10 秒一次，自动说话 30 秒冷却，防止过度调用
4. **兼容性**：当前版本仅支持 Minecraft 1.20.1 和 Forge 47.3.0
5. **性格设定**：性格不再由数值定义，改为在 `system_prompt` 中用文字描述
